## 2024-05-24 - [Arbitrary URL Opening & RCE Risk]
**Vulnerability:** Arbitrary protocol opening via `shell.openExternal`. The `IPC.OPEN_EXTERNAL` handler and `openUrlInBrowser` interceptor directly passed unsanitized user/frontend input to the OS shell, allowing dangerous protocols like `file://` or `javascript:`.
**Learning:** The Electron main process relies completely on manual validation for OS-level actions. Also, `shell.openExternal` returns a `Promise`, making synchronous return-value checking (as previously done for Jira URLs) invalid. Testing for Electron modules requires `node:test` executed via `npx ts-node -T` instead of project-level vitest to bypass dev-only compilation errors in production configs.
**Prevention:** Implement a central, strict URL validation utility (`isValidExternalUrl`) allowing only safe protocols (`http:`, `https:`, `mailto:`). All `shell.openExternal` calls should route through an abstraction (`openExternalUrl`) that enforces this validation before delegating to the OS.
