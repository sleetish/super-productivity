## 2024-05-24 - Secure random string generation with rejection sampling
**Vulnerability:** Weak identifier generation and modulo bias. Using `Math.random()` to generate identifiers from a fixed character set is not cryptographically secure, and a naive modulo operation introduces modulo bias (certain characters appearing more frequently than others).
**Learning:** `Math.random()` should be replaced by `window.crypto.getRandomValues()` (with fallback to `globalThis.crypto` or `globalThis.webcrypto` for environments like SSR or Electron). When generating a random string from a fixed set of characters (e.g. Base62), rejection sampling is necessary to maintain uniform distribution and eliminate modulo bias. For example, when picking an index from a 62-character string using random 8-bit values (0-255), values >= 248 must be rejected because 256 is not evenly divisible by 62 (256 = 62 * 4 + 8).
**Prevention:** Use `crypto.getRandomValues()` and implement rejection sampling when taking the modulo of random values against an array length that doesn't evenly divide the maximum value of the random integer type. Also, allocate a typed array (like `Uint8Array`) upfront for performance.
